🚨 OFFICIAL DIRECTIVE FOR DEV & UI/UX TEAMS 🚨
SUBJECT: Complete Replacement of the App’s Core Content Architecture
To the Development and Design Teams:
Please immediately DEPRECATE the generic "Core Content Modules" (Section 5.2 and Pages 6, 7, & 8 of the previous UI plan).
We are pivoting to a highly structured, scientifically backed learning taxonomy. Every single book processed into this app must now strictly follow the hierarchical tree below. Additionally, we are introducing a new core feature: Tap-Designed Chapter Summaries (operating like Instagram Stories).
All database schemas, JSON structures, and UI components must be rebuilt to support this exact flow.
1. THE NEW BACKEND ARCHITECTURE (The Database Schema)
Backend Developers: The JSON/Database object for every "Book" must now be structured exactly like this hierarchical tree:
📦 BOOK OBJECT
├── 🎯 Core Thesis: (String: The overarching 1-2 sentence "Big Idea" of the whole book)
├── 📑 Chapter Summaries: (Array: A dedicated "tap-through" breakdown for each specific chapter)
├── 🧠 Mental Models: (Array of Objects: Reusable visual frameworks and systems)
├── 📜 Principles: (Array of Objects: The fundamental rules to live/work by)
│   └── 📖 Stories: (Nested Object: The real-world anecdote that proves its parent principle)
├── 🚫 Common Mistakes: (Array: Anti-patterns / what the user is currently doing wrong)
├── ✍️ Exercises: (Array: Interactive forms, dynamically ranked by an "Impact" score)
└── ✅ Action Items: (Object: Categorized strictly into immediate vs. long_term)
2. UPDATED PAGE 6: The Book Master Hub (UI/UX)
Purpose: This replaces the old book detail page. When a user clicks a book, they don't just see a generic "Read Summary" button. They see this highly organized psychological syllabus, allowing them to extract exactly what they need today.
• How it looks (UI/UX):
• The Hero Header: Book Cover, Title, and a massive, bold text block displaying the 🎯 Core Thesis. The user sees the ultimate bottom-line of the book before they even scroll or click anything.
• The Blueprint Grid (The Navigation): Below the thesis, the screen features a beautifully designed grid of clickable UI tiles mapping exactly to the new structure:
• [ 📑 Tap-Through Chapter Summaries ] (Featured prominently at the top)
• [ 🧠 Mental Models ]
• [ 📜 Principles & Stories ]
• [ 🚫 Common Mistakes ]
• [ ✍️ Exercises ]
• [ ✅ Action Items ]
• Functional Features: Tapping any of these tiles launches the "Interactive Engine" (Page 7) but only loads the specific data format the user selected.
3. UPDATED PAGE 7: The "Interactive Engine" (The New UI Cards)
Frontend Developers: The UI where the user actually consumes the data operates on a Tap-Designed interface (like Instagram Stories or Duolingo). The user taps the right edge of the screen to advance. You must build distinct UI templates for these new categories:
• Template 1: 📑 "Tap-Designed" Chapter Summaries (The New Feature)
• UI/UX: The user selects "Chapter 1." The screen fills with a fast-paced visual journey. A segmented progress bar sits at the top.
• Function: Each screen card contains just 1 or 2 punchy, highly readable sentences summarizing a key event from that chapter. The user physically taps the screen to advance through the chapter's narrative. At the end of Chapter 1, a smooth animation asks: "Start Chapter 2?"
• Template 2: 🧠 "Mental Models" (The Visual Frameworks)
• UI/UX: Replaces text with interactive, full-screen infographics (e.g., The Eisenhower Matrix, CBT Triangles).
• Function: Tap-to-Reveal. The diagram starts partially blank. As the user taps the screen, pieces of the framework animate in one by one, explaining how the mental model applies to real life step-by-step.
• Template 3: 📜 "Principle + Story" (The Parent/Child Flip-Card)
• UI/UX: A two-part interactive card sequence to ensure the user always gets the context.
• Function (Tap 1): The front of the card displays a bold Principle (e.g., "Rule 1: Environment design matters more than willpower").
• Function (Tap 2): At the bottom, a pulsing button says "👀 See the Proof (Read Story)". Tapping it flips the card over in 3D (or slides it up) to reveal the Extracted Story (the historical or business anecdote that proves the principle works).
• Template 4: 🚫 "Common Mistakes" (Anti-Patterns)
• UI/UX: A "Do This / Not That" split-screen or swipe card.
• Function: Shows the flawed way people usually think (Red/Mistake) juxtaposed directly next to the psychological correction taught in the book (Green/Solution).
4. UPDATED PAGE 8: The Action Engine (Execution)
Developers: This handles the bottom of the data tree. This is where the user actively applies the psychology to their life.
• Template 5: ✍️ Exercises (Ranked by Impact)
• UI/UX: A digital workbook interface with auto-expanding text boxes where users type their reflections.
• Function (Sorting Algorithm): Exercises are dynamically tagged in the database by impact. The UI will display prominent badges next to each prompt: [🔥 High Impact], [⚡ Medium Impact], or [🌱 Low Impact].
• Logic: The app forces the "High Impact" exercises to the top of the screen, ensuring busy users do the most life-changing mental work first before getting fatigued.
• Template 6: ✅ Action Items (Immediate vs. Long-Term)
• UI/UX: A dynamic checklist interface with a master toggle switch at the top: [ Immediate (Do Today) ] <----O----> [ Long-Term (Habits) ].
• Function (Immediate): Quick 5-minute tasks (e.g., "Write down 3 triggers for your bad habit right now"). Users check a box, which crosses out the text, buzzes the phone (haptic feedback), and plays a rewarding "ding" sound.
• Function (Long-Term): Broader lifestyle changes (e.g., "Track your sleep hours for 30 days"). Includes a button next to the item that says [🗓️ Add to Calendar], which opens the native iOS/Android API to schedule the task in the user's personal phone calendar application.
