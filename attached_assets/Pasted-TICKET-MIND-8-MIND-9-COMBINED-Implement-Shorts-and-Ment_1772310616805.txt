3) Add proper foreign key relations and indexes on book_id and chapter_id for both tables.
4) Run the database migration to apply changes.

PART B — API ENDPOINTS:
Customer-facing (public):
- GET /api/books/:id/shorts — list all published shorts for a book
- GET /api/chapters/:id/shorts — list shorts for a specific chapter
- GET /api/books/:id/mental-models — list all published mental models for a book
- GET /api/chapters/:id/mental-models — list mental models for a specific chapter

Admin (auth required):
- POST /api/admin/shorts — create a new short
- PUT /api/admin/shorts/:id — update a short
- DELETE /api/admin/shorts/:id — delete a short
- POST /api/admin/mental-models — create a new mental model
- PUT /api/admin/mental-models/:id — update a mental model
- DELETE /api/admin/mental-models/:id — delete a mental model
